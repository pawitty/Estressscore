Why This Problem Matters

  • 🔥 1000+ heat-related deaths annually
  • 👥 Crowd density increases stampede risk exponentially
  • ⚠️ Most disasters happen due to lack of early detection
  • 📉 Current systems are reactive, not predictive

Problem Statement

  • High-density crowds during elections
  • Extreme heat + humidity
  • Leads to stampedes, panic, heat strokes
  • CCTV → reactive, expensive

Gap in Existing Systems

  • No quantifiable stress metric
  • No predictive intelligence
  • No prioritized response system

Our Solution

  • Multi-sensor system
  • Real-time stress score (0–1)
  • Level-based intervention system

Stress Score Model

S = w₁·Air + w₂·Temp + w₃·Humidity + w₄·Sound + w₅·Consensus

  • Normalized inputs
  • Weighted importance
  • Clamped between 0 and 1

Weight Logic

  • Sound → fastest crowd stress signal
  • Temperature → heat danger
  • Humidity → amplifies stress
  • Air quality → long-term health impact
  • Consensus → reduces false alarms

Signal Processing

  • Sensor data is noisy
  • Use Exponential Moving Average (EMA)
  • Removes sudden spikes

Stress Levels & Actions

  • 🟢 0–0.4 → Normal
  • 🟡 0.4–0.65 → Monitor
  • 🟠 0.65–0.85 → Deploy resources
  • 🔴 0.85–1 → Emergency

Hardware Prototype

  • ESP32 microcontroller
  • Temperature & humidity sensors
  • Microphone (sound levels)
  • Air quality sensor

System Architecture

  • Edge: Sensors + ESP32 + local compute
  • Primary: ESP-NOW mesh (fast)
  • Fail-safe: LoRa backup
  • Cloud: Dashboard + OTA updates

Mesh Networking (ESP-NOW)

  • Node-to-node communication
  • No single point of failure
  • Consensus improves accuracy

LoRa Fail-Safe

  • ESP-NOW handles normal communication
  • If mesh fails → LoRa activates
  • Only sends critical alerts

System remains operational during failure

Adaptive Intelligence (OTA)

  • Weights updated dynamically
  • Adjusts to environment
  • Heatwave → temp weight ↑
  • Indoor → sound weight ↑

System Flow

Sensors → Stress Score → Mesh → EMA → Threshold → Action → LoRa → Dashboard → OTA

Dashboard

  • Live stress visualization
  • Heatmaps
  • Transparency for authorities

Failure Handling

  • Internet down → LoRa works
  • Node failure → mesh reroutes
  • Noise spikes → EMA smooths
  • Calibration errors → OTA fixes
